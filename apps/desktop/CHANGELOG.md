# Changelog — SwaggerMan Desktop

크로스플랫폼(Windows·macOS·Linux) OpenAPI 탐색기. Tauri 2 + React + TypeScript.

릴리스 빌드(설치본)는 `SwaggerMan-v*` 태그 푸시 시 GitHub Actions가 생성하며,
**Releases** 페이지에서 OS별 설치본을 내려받을 수 있습니다.

> **macOS 설치 시 "손상되어 열 수 없습니다"가 뜨면** (서명 미인증 앱): 터미널에서
> `xattr -dr com.apple.quarantine /Applications/SwaggerMan.app` 실행 후 다시 여세요.

## v0.4.16

- 안정화: **Mock 프리셋 저장(IndexedDB) 회귀 테스트 보강** — 실세계 형태(FastAPI·`/api/v1` 경로) 스펙에서 브라우저 캡처 → 전체 저장 → IndexedDB 영속 → 재로드 전 과정을 검증하는 테스트를 추가했습니다. (프리셋 저장·선택·삭제 동작은 v0.4.15와 동일)

## v0.4.15

- 버그수정: **대용량 캡처를 Mock 프리셋으로 저장하면 안 저장되던 문제** — 실제 API 응답(CMDB 등)은 커서 localStorage(약 5MB) 용량을 초과하면 저장이 조용히 실패했는데 "저장됨" 메시지만 떠서, Mock 화면에 프리셋이 안 보였습니다. **프리셋 저장소를 IndexedDB(대용량)로 옮겨** 큰 캡처도 저장되며(기존 프리셋은 자동 이전), 저장에 실패하면 정직하게 에러를 표시합니다

## v0.4.14

- 변경: **프록시 "전체 Mock으로"는 제목 붙인 Mock 프리셋으로 저장**한다. 저장 후 **Mock 서버 모달의 프리셋 드롭다운에서 그 제목을 선택**하면 적용됩니다(현재 활성 설정은 자동으로 바꾸지 않음). 저장 결과 메시지에 "프리셋을 선택해 적용하세요" 안내를 추가했습니다

## v0.4.13

- 버그수정: **프록시 "전체 Mock으로"가 Mock 서버 목록에 반영되지 않던 문제** — v0.4.12에서 전체저장이 프리셋으로만 저장되고 활성 Mock 설정에는 적용되지 않아, Mock 서버 모달을 열어도 저장한 내용이 목록에 보이지 않았다. 이제 **활성 설정에 바로 적용**하면서 **제목 프리셋으로도 함께 저장**한다(둘 다 반영)

## v0.4.12

- 기능: **Mock 프리셋** — 현재 Mock 설정(전 operation의 dataset/body/켜짐)을 **제목 붙여 여러 개 저장**하고 드롭다운으로 골라 불러온다(불러올 때 확인 후 현재 설정 덮어쓰기). 시나리오별 응답 세트를 오가며 쓸 수 있다. 스펙별로 보관되며 이름변경·삭제 가능
- 기능: **프록시 "전체 Mock으로"에 제목 입력** — 녹화 전체를 제목 붙인 **새 Mock 프리셋**으로 저장한다(현재 활성 Mock 설정은 건드리지 않음). 저장 후 Mock 서버에서 그 프리셋을 골라 쓰면 된다

## v0.4.11

- 기능: **브라우저 캡처(CDP)** — 프록시 녹화의 '브라우저' 모드. 전용 Chrome을 디버깅 모드로 띄워 그 안의 API 호출(XHR/fetch)을 직접 녹화한다. **Okta 등 OAuth 로그인은 인증 후 다른 호스트로 리다이렉트되어 리버스 프록시를 우회**하는데, 브라우저 자체에 붙으므로 오리진이 바뀌어도 로그인 뒤 트래픽을 모두 잡는다. 녹화 항목은 기존처럼 **Mock으로/전체 Mock으로** 저장 가능. 전용 프로필이라 한 번 로그인하면 다음 녹화에 세션이 재사용된다(v1은 첫 탭만 캡처 — 별도 팝업창 SSO는 미지원)

## v0.4.10

- 기능: **프록시 녹화 "전체 Mock으로"** — 녹화 목록 전체를 버튼 한 번으로 Mock 설정에 일괄 저장한다. 같은 API가 여러 번 녹화됐으면 최신 응답이 저장되고, 스펙에 없는 경로·실패한 녹화는 제외 건수로 알려준다
- 기능: **컬렉션 저장 요청 수정** — 컬렉션 목록에서 ✏️로 이름·메서드·URL을 그 자리에서 고친다(http→https 같은 URL 교정). 헤더·바디까지 바꾸려면 불러와서 수정한 뒤 **"불러온 요청에 덮어쓰기"** 로 원본에 반영
- 기능: **프록시가 쿠키/리다이렉트 인증(Okta 등 SSO)을 통과시킴** — Set-Cookie·Location 등 응답 헤더를 전달하고(localhost용 재작성), 리다이렉트를 클라이언트가 직접 따라가게 하며, 쿠키 포함 요청의 CORS(preflight 포함)를 처리한다. 설정의 **SSL 검증 끄기·프록시·타임아웃이 프록시 포워딩에도 적용**돼 사내망(사설 CA)에서 https 타깃이 502 나던 문제 해결
- 수정: **Base URL을 바꿔도 재실행하면 원래대로 돌아가던 문제** — 직접 입력하거나 환경으로 바꾼 Base URL을 프로젝트별로 저장해 앱 재시작·프로젝트 전환 후에도 유지한다. 입력란 옆 ↺ 버튼으로 스펙 기본값 복원

## v0.4.9

- 수정: **컬렉션 러너가 첫 요청만 실행하고 멈추던 문제** — 저장된 요청 하나가 실행 불가 상태(옛 버전 저장 데이터 등)면 그 자리에서 러너가 멈추고 "실행 중…"으로 잠겨 재실행도 불가했다. 이제 실패한 요청은 **오류**로 표시하고 나머지를 계속 실행하며, 실행이 끝나면 버튼이 항상 복구된다
- 기능: **러너 반복 실행** — 실행 버튼 옆 반복 횟수(N)를 지정하면 컬렉션 전체를 N회 연속 실행한다. 통과 배지는 누적 집계(예: 요청 2개 × 3회 = 6/6 통과)
- 개선: **컬렉션 저장 기본 이름 = URL 경로** — 요청 이름을 비워두고 저장하면 "GET 요청" 대신 URL 경로(예: `/api/v1/stacks`)가 이름이 된다. 입력란에도 미리 표시돼 무엇이 저장될지 보인다

## v0.4.8

- 기능: **파일에서 프로젝트 가져오기** — 로컬 OpenAPI 스펙 파일(JSON/YAML)을 **프로젝트 관리(✏️) → "파일에서 가져오기"** 로 바로 불러온다. 파일 내용을 앱에 스냅샷으로 복사해 보관하므로 원본 파일을 옮기거나 지워도 프로젝트가 그대로 열리고, 앱 재실행 시 자동 복원된다. 가져온 프로젝트는 목록에 📄 파일명으로 표시되며, 파일이 바뀌면 **"다시 가져오기"** 로 히스토리를 유지한 채 갱신할 수 있다(URL 없이 오프라인·사내망 스펙도 탐색 가능)

## v0.4.7

- 수정: **후원(커피) 배너가 리로드/재실행마다 다시 뜨던 문제** — 기존엔 '닫기(✕)'를 눌러야만 하루 동안 억제됐고, 닫지 않고 리로드하면 매번 다시 표시됐다. 이제 배너가 **표시되는 순간 그 시각을 기록**해, 닫지 않아도 하루(24시간) 내에는 다시 뜨지 않는다(v0.4.6의 보완)

- 기능: **프록시 녹화 모드** — 실서버 앞에 로컬 프록시(`localhost:9091`)를 두고 트래픽을 투명 포워딩하면서 요청/응답을 자동 녹화. 녹화 항목을 "Mock으로 보내기"로 경로 매칭되는 API의 Mock 데이터로 변환

첫 릴리스. macOS 네이티브 앱(SwiftUI)의 핵심 기능을 크로스플랫폼으로 이식.

### 스펙 / 요청
- OpenAPI 3.x · Swagger 2.0 로드 (JSON/YAML, 로컬 `$ref` 해석)
- Swagger UI(`index.html`) 자동 디스커버리 (well-known 경로 + `swagger-config`, 401 후보 회피)
- HTTP 요청은 Rust(reqwest)로 처리 — 웹뷰 CORS/스코프 제약 없이 임의 호스트 호출
- 요청 편집: path/query/header/body, 요청 URL 미리보기(한글 raw 표시)
- body 미리 채우기: 스펙 `example` 우선, 없으면 스키마로 예시 생성
- body 샘플 저장/수정/삭제/선택 (오퍼레이션별)
- 요청 취소, 전송 중 로딩 표시

### 인증 / 환경 / 헤더
- Authorize: 보안 스킴(bearer/basic/apiKey)별 토큰 → 요청 헤더 자동 적용
- 환경(여러 Base URL) 관리 모달 (추가/수정/삭제/적용)
- 전역 헤더(모든 요청 적용) 관리 모달 + 요청 화면 읽기 전용 표시

### 응답
- Docs/Response 탭 (파라미터·요청/응답 스키마 트리)
- 라인 번호 + JSON 구문 색상, 미니맵(뷰포트·검색 매치·클릭 이동)
- 검색(Enter, 매치 내비게이션), Body/cURL/코드 스니펫(curl/JS/Python) 복사

### 탐색 / 영속화
- 사이드바: 검색·메서드·태그 필터, 즐겨찾기(★)
- 히스토리: 요청값+응답 저장, 클릭 복원/replay/삭제, 선택 표기 배너
- 프로젝트(spec URL) 목록 저장·전환, 시작 시 마지막 spec 자동 로드
- 프로젝트별 영속화(즐겨찾기/히스토리/인증/환경/전역헤더/샘플), 전역 줌

### 3-pane / UI
- react-resizable-panels 드래그 리사이즈(크기 저장), 다크 테마
- 모던 select·세그먼트 탭·라인 아이콘

### 품질
- 코어 로직 단위 테스트(vitest) 44개, ESLint(정적 분석) + 타입체크 무결
- GitHub Actions CI(lint/typecheck/test/build/cargo) 및 릴리스 워크플로(mac/win 빌드)
