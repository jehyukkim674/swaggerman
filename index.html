1. 소개

SwaggerMan(스웨거맨)은 OpenAPI(Swagger) 명세를 불러와 API를 한눈에 탐색하고, 실제 요청을 보내 응답을 확인하는 크로스플랫폼 데스크톱 앱입니다. macOS(Apple Silicon)와 Windows를 지원하며, Tauri 2 · React · TypeScript로 만들어졌습니다.

이 앱의 가장 큰 차별점은 로컬에 설치된 claude CLI와 연동되는 AI 어시스턴트(✦ AI)입니다. 현재 보고 있는 엔드포인트를 컨텍스트로 한국어 대화를 나누고, 자연어로 요청 폼을 자동으로 채우거나 응답을 설명·진단받을 수 있습니다.

이런 분께 유용합니다.

  • 백엔드/프론트엔드 개발자 — API 동작을 빠르게 확인하고 테스트
  • QA 엔지니어 — 다양한 파라미터·상태코드 검증, 요청 이력 비교
  • API를 처음 다루는 사람 — AI 도움으로 요청을 쉽게 구성

이 매뉴얼의 스크린샷에는 1 2 같은 빨간 번호가 표시되어 있고, 본문 설명이 그 번호와 1:1로 대응합니다.

2. 설치

설치 파일은 GitHub Releases 페이지에서 운영체제에 맞게 내려받습니다.

macOS (Apple Silicon)

  1. Releases에서 .dmg 파일을 내려받아 엽니다.
  2. SwaggerMan 아이콘을 Applications 폴더로 드래그합니다.
  3. 응용 프로그램에서 SwaggerMan을 실행합니다.

첫 실행 시 “손상되어 열 수 없습니다” 또는 “개발자를 확인할 수 없습니다”가 뜰 수 있습니다(서명 미인증 앱). 우클릭 → 열기로 안 되면 터미널에서 아래를 실행한 뒤 다시 여세요.

xattr -dr com.apple.quarantine /Applications/SwaggerMan.app

Windows

  • .exe — 설치형 인스톨러(NSIS). 실행 후 안내에 따라 설치합니다.

다운로드 시 SmartScreen 경고(“Windows의 PC를 보호했습니다”)가 뜨면 추가 정보 → 실행을 누르세요(터미널 작업 불필요).

설치 후 시작 메뉴에서 SwaggerMan을 실행할 수 있습니다.

3. 화면 구성

SwaggerMan의 화면은 크게 다섯 영역으로 나뉩니다.

  • 1 상단바 — 프로젝트(스펙)·환경·인증·각종 도구 버튼이 모여 있습니다. 아래에서 자세히 설명합니다.
  • 2 사이드바 — 불러온 스펙의 엔드포인트 목록. 검색·메서드 필터·태그 필터·즐겨찾기(☆)와 히스토리 탭이 있습니다.
  • 3 요청 편집기 — 선택한 엔드포인트의 파라미터·헤더·Body를 입력하고 전송합니다.
  • 4 Docs / Response 패널 — 스펙 문서(파라미터·스키마)와 응답 결과를 탭으로 전환하며 봅니다.
  • 5 ✦ AI 패널 — 현재 엔드포인트를 컨텍스트로 AI와 대화하고 요청 폼을 자동 작성합니다. » 버튼으로 접을 수 있습니다.

상단바 자세히 보기

  • 1 프로젝트 선택·관리(✏️) — 저장된 프로젝트(스펙)를 드롭다운으로 전환하고, ✏️로 목록을 관리합니다.
  • 2 스펙 URL 입력란 — OpenAPI 스펙 주소를 입력합니다. (예: https://api.example.com/v3/api-docs)
  • 3 Load — 입력한 스펙을 불러옵니다. 입력란에서 Enter로도 가능합니다.
  • 4 cURL — cURL 명령을 붙여넣어 요청으로 가져옵니다.
  • 5 컬렉션 — 요청을 저장·관리하고 Postman 컬렉션을 가져오거나 내보냅니다.
  • 6 러너 — 컬렉션의 요청을 일괄 실행하고 통과/실패 리포트를 봅니다.
  • 7 새 창 — 새 창(⌘N)을 열어 다른 프로젝트를 동시에 봅니다.
  • 8 설정(⚙) — 네트워크(타임아웃·SSL·프록시), claude 경로, 쿠키를 관리합니다.
  • 9 Base URL · 환경 전환 — 요청을 보낼 서버 주소와 환경을 선택합니다.
  • 10 환경 관리 — 개발/운영 같은 환경과 변수를 추가·수정합니다.
  • 11 전역 헤더 설정 — 모든 요청에 자동으로 붙는 헤더를 관리합니다.
  • 12 Authorize — 보안 스킴(Bearer/Basic/API Key/OAuth2) 토큰을 설정합니다. 적용 개수가 표시됩니다.
  • 13 업데이트 확인 — 새 버전을 확인하고 인앱에서 설치합니다.
  • 14 ✦ AI — AI 패널을 열고 닫습니다.
  • 15 테마 전환(☀︎/☾) — 다크/라이트 테마를 전환합니다. 옆의 −/+로 화면 배율(줌)을 조절합니다.

4. 시작하기

스펙 불러오기

  1. 상단바 입력란(2)에 OpenAPI 스펙 URL을 입력합니다. JSON/YAML의 OpenAPI 3.x · Swagger 2.0을 지원하며, Swagger UI 주소 (…/swagger-ui/index.html)를 넣으면 스펙을 자동 탐색합니다.
  2. Load(3)를 누릅니다. 로딩 중에는 전체 화면 오버레이로 진행 상태가 표시됩니다.
  3. 사이드바에 엔드포인트가 태그별로 나타나면 항목을 클릭해 요청 편집기를 엽니다.

불러온 스펙은 프로젝트로 자동 저장됩니다. 즐겨찾기·히스토리·인증·환경·샘플이 모두 프로젝트별로 보관되며, 앱을 재시작하면 마지막 프로젝트와 마지막으로 보던 API·입력값이 그대로 복원됩니다.

사내 CA·자체 서명 인증서 서버의 스펙이 error sending request로 안 열리면, 설정(⚙) → “SSL 인증서 검증 무시”를 켠 뒤 다시 Load 하세요. (요청 실행·OAuth2 토큰 발급에도 동일하게 적용됩니다.)

프로젝트 관리

상단바 ✏️(1)를 누르면 저장된 프로젝트를 한곳에서 관리하는 모달이 열립니다.

  • 1 프로젝트 목록 — 이름과 스펙 URL을 바로 수정할 수 있습니다. 수정 내용은 입력 즉시 저장됩니다.
  • 2 열기 — 해당 프로젝트의 스펙을 다시 불러옵니다. URL을 바꾼 뒤에는 열기로 다시 로드하세요.
  • 3 삭제 — 프로젝트를 목록에서 제거합니다(히스토리·즐겨찾기 포함).
  • 4 새 프로젝트 추가 — 이름(생략 가능)과 스펙 URL만 입력하면 목록에 추가됩니다.

새 창(멀티윈도우)

상단바 새 창(7) 또는 ⌘N으로 창을 여러 개 열 수 있습니다. 창마다 다른 프로젝트를 선택해 여러 API를 동시에 볼 때 유용합니다. 창 생성이 실패하면 원인이 상단바에 표시됩니다.

5. 요청 보내기

사이드바에서 엔드포인트를 선택하면 요청 편집기가 열립니다. 스펙에 정의된 파라미터는 입력 칸이 자동으로 만들어지고, 예시 값(example/default/enum)이 미리 채워집니다.

  • 1 메서드·경로·전송 — ↺ 초기화로 파라미터를 스펙 기본값으로 되돌리고, Send(또는 ⌘Enter)로 요청을 보냅니다.
  • 2 요청 URL 미리보기 — 실제 전송될 URL을 실시간으로 보여줍니다. 긴 URL도 잘리지 않게 여러 줄로 표시되고, 옆의 복사 버튼으로 전체를 복사합니다.
  • 3 요청 샘플 — 현재 Query·Headers·Body 묶음을 이름 붙여 저장하고 드롭다운으로 전환합니다. 같은 API를 여러 케이스로 테스트할 때 유용합니다. (GET 요청도 가능)
  • 4 Query Params — 쿼리 파라미터. 필수 표시와 타입 힌트가 붙습니다. 체크박스로 개별 파라미터를 켜고 끌 수 있습니다.
  • 5 Headers — 요청 헤더. 값에 {{변수}}를 쓰면 전송 시 현재 환경의 값으로 치환됩니다. (스크린샷의 {{apiToken}} 참고)
  • 6 테스트 & 체이닝 — 응답 값 추출 규칙과 어서션(검증)을 정의합니다. (8. 환경과 변수 참고)

Body 입력 (POST/PUT)

  • 1 Headers — Body가 있는 요청은 Content-Type이 자동으로 추가됩니다.
  • 2 Body 에디터 — 스펙의 example(없으면 스키마 기반 예시)이 미리 채워지며, JSON 구문 색상이 적용됩니다. 형식은 JSON 외에 form-urlencoded·multipart/form-data(파일 업로드)도 지원합니다.
  • 3 정렬 · 압축 — JSON을 보기 좋게 정렬하거나 한 줄로 압축합니다.
  • 4 요청 샘플 — Body를 포함한 묶음을 샘플로 저장/전환합니다.

cURL 가져오기

상단바 cURL(4) 버튼에 cURL 명령을 붙여넣으면 메서드·URL·헤더·Body가 분석되어 요청으로 만들어집니다. 다른 도구나 문서에 있는 요청을 빠르게 재현할 때 씁니다.

전송 전 사전 검증 — 필수 path/query/body가 비어 있으면 전송 전에 경고를 표시해 실수를 막아 줍니다.

6. 응답 보기

요청을 보내면 우측 패널의 Response 탭에 결과가 나타납니다.

  • 1 상태코드 · 소요 시간 · 크기 — 한눈에 결과를 확인합니다.
  • 2 보기 모드 · 내보내기 — Pretty(정렬+구문 색상) / Raw(원본) 전환, 저장(파일로), cURL·Code ▾ (JavaScript·Python 등 코드 스니펫) 복사, ✦ 설명(AI가 응답을 해설).
  • 3 스키마 검증 — 응답 본문을 스펙의 응답 스키마와 자동 대조해 불일치(누락 필드·타입 오류)를 표시합니다.
  • 4 검색 — ⌘F로 포커스, Enter로 다음 매치 이동. 매치는 본문과 미니맵에 하이라이트됩니다.
  • 5 응답 본문 — 라인 번호 + JSON 구문 색상. 10MB가 넘는 대용량 응답도 가상 스크롤로 멈춤 없이 표시됩니다.
  • 6 미니맵 — 전체 본문에서 현재 위치와 검색 매치 위치를 보여줍니다. 클릭하면 그 위치로 이동합니다.

Docs 탭

같은 패널의 Docs 탭에서는 선택한 엔드포인트의 스펙 문서를 봅니다.

  • 1 엔드포인트 정보 — 메서드·경로·설명.
  • 2 PARAMETERS — 파라미터 이름·위치(query/path)·타입·필수 여부. enum 값("available" | "pending" | "sold")도 표시됩니다.
  • 3 RESPONSES — 상태코드별 응답 스키마 트리. ▸를 눌러 중첩 객체를 펼치며 필드 타입·필수 여부를 확인합니다. 스펙에 example 응답이 있으면 함께 표시됩니다.

7. 인증 (Authorize)

스펙에 보안 스킴(security scheme)이 정의되어 있으면 상단바의 Authorize(12) 버튼으로 토큰을 설정합니다. 적용된 토큰은 요청을 보낼 때 자동으로 헤더에 포함됩니다.

  • 1 OAuth2 스킴 — 발급받은 액세스 토큰을 직접 붙여넣거나, 아래 3으로 자동 발급해 적용합니다.
  • 2 API Key 스킴 — 키 값을 입력하고 Authorize를 누르면 적용됩니다. 적용된 값은 가려져(•••) 표시되며 보기/숨김을 전환할 수 있고, Logout으로 해제합니다.
  • 3 OAuth2로 토큰 발급 — Grant 방식(client_credentials/password), Token URL, Client ID/Secret, Scope를 입력하고 토큰 발급을 누르면 토큰을 받아 선택한 스킴에 자동 적용합니다.
  • 4 모두 저장 — 입력한 토큰들을 일괄 적용합니다.

Bearer/Basic/API Key 등 스킴 종류에 맞는 헤더(Authorization: Bearer …, api_key: … 등)가 자동으로 구성됩니다. 상단바 Authorize 버튼에 적용 개수(예: 1/2)가 표시됩니다.

8. 환경과 변수

개발·스테이징·운영처럼 서버를 오가며 테스트할 때는 환경을 만들어 두고 상단바에서 전환하세요. 환경마다 Base URL과 변수를 가질 수 있습니다.

  • 1 환경 이름 · Base URL — 환경의 이름(개발/운영 등)과 요청을 보낼 서버 주소.
  • 2 변수 목록 — 이 환경에서 쓸 변수들. 요청의 어느 입력 칸에서든 {{petId}}, {{apiToken}}처럼 참조합니다.
  • 3 적용 — 이 환경을 현재 환경으로 선택합니다. 상단바 드롭다운에서도 전환할 수 있습니다.
  • 4 두 번째 환경 — 같은 변수 이름에 환경마다 다른 값을 넣어두면, 환경만 바꿔서 같은 요청을 다른 서버·계정으로 보낼 수 있습니다.
  • 5 + 환경 추가 — 새 환경을 만듭니다.

{{변수}} 사용

  • URL·path/query 파라미터·헤더·Body 어디서든 {{이름}} 형태로 사용합니다.
  • 값 입력 중 {{를 입력하면 자동완성 목록이 떠서 변수를 고를 수 있습니다.
  • 입력된 {{변수}} 위에 마우스를 올리면 툴팁으로 출처(환경/체이닝/동적)와 실제 치환 값이 표시됩니다.

동적 변수

매번 바뀌는 값이 필요할 때는 내장 동적 변수를 사용합니다.

변수값
{{$timestamp}}현재 Unix 타임스탬프(초)
{{$isoTimestamp}}ISO 8601 형식 현재 시각
{{$guid}} / {{$randomUUID}}랜덤 UUID
{{$randomInt}}랜덤 정수

요청 체이닝과 어서션

요청 편집기의 테스트 & 체이닝 섹션(5장 6)에서 설정합니다.

  • 추출 규칙(체이닝) — 응답에서 JSONPath로 값을 뽑아 변수에 저장합니다. 예: 로그인 응답의 $.accessToken → {{accessToken}} 변수로 저장 → 다음 요청 헤더에서 사용.
  • 어서션 — 상태코드나 JSONPath 값(같음/포함/존재)으로 응답을 자동 검증하고 통과/실패를 표시합니다.

9. 히스토리와 비교

보낸 요청은 응답과 함께 히스토리에 자동 저장됩니다. 사이드바의 히스토리 탭에서 확인하세요.

  • 1 히스토리 탭 — 누적 개수가 함께 표시됩니다.
  • 2 비교 · 전체 삭제 — 체크박스로 2건을 고르면 비교 (2/2) 버튼이 활성화됩니다.
  • 3 히스토리 목록 — 메서드·경로·상태코드·시각이 표시됩니다. 항목을 클릭하면 당시 요청 값과 응답이 그대로 복원되어 재전송(replay)할 수 있습니다.

두 요청 비교

“같은 API인데 왜 결과가 다르지?”를 확인할 때 사용합니다. 히스토리에서 2건을 선택하고 비교를 누르세요.

  • 1 요청 정보 비교 — 두 요청(A/B)의 URL·상태코드·실행 시각·소요 시간을 나란히 봅니다.
  • 2 파라미터 diff — path/query/header를 키별로 비교해 다른 값을 강조합니다.
  • 3 응답 본문 diff — 줄 단위 비교. + 추가 / − 삭제 / ~ 변경(변경 전·후 쌍)으로 색이 구분됩니다. 대용량 응답도 가상 스크롤로 부드럽게 동작합니다.
  • 4 diff 검색 · 범례 — 검색어를 입력하면 매치 위치로 이동하며 파란색으로 강조됩니다.
  • 5 미니맵 — 전체 diff에서 추가/삭제/변경 위치를 색으로 보여줍니다.

10. 컬렉션과 러너

자주 쓰는 요청은 컬렉션으로 저장해 두세요. 스펙과 무관하게 보관되며, Postman 컬렉션도 가져올 수 있습니다.

  • 1 가져오기 · 내보내기 — Postman 컬렉션 v2.1 또는 SwaggerMan 네이티브 JSON을 가져오고, 네이티브 JSON으로 내보냅니다.
  • 2 현재 요청 저장 — 지금 편집 중인 요청을 이름을 붙여 선택한 컬렉션에 저장합니다.
  • 3 컬렉션 · 저장된 요청 목록 — 컬렉션별로 요청이 정리됩니다.
  • 4 불러오기 — 저장된 요청을 요청 편집기로 불러옵니다.

컬렉션 러너 (일괄 실행)

상단바 러너(6)로 컬렉션의 요청을 한 번에 실행하고 결과를 확인합니다. 스모크 테스트나 회귀 확인에 유용합니다.

  • 1 컬렉션 선택 — 실행할 컬렉션을 고릅니다.
  • 2 실행 · 통과 리포트 — 실행 후 전체 통과/실패 수가 표시됩니다.
  • 3 요청별 결과 — 각 요청의 상태코드·소요 시간·성공 여부(✓/✗). 어서션을 정의해 두면 어서션 결과까지 반영됩니다.

11. ✦ AI 어시스턴트

SwaggerMan의 핵심 기능입니다. 우측 ✦ AI 패널에서, 지금 보고 있는 엔드포인트를 컨텍스트로 로컬 claude CLI와 한국어로 대화합니다.

  • 1 모델 선택 · 새 대화 — Sonnet(균형)/Opus(고성능)/Haiku(빠름) 중 선택. 새 대화로 컨텍스트를 초기화합니다.
  • 2 대화 — 질문하면 답변이 스트리밍으로 표시됩니다. 엔드포인트의 파라미터·스키마를 알고 있어 구체적으로 답합니다.
  • 3 요청 제안 카드 — /요청 …으로 자연어 지시를 하면 AI가 path·query·headers·body를 제안합니다.
  • 4 폼에 적용 · cURL 복사 · 변수로 저장 — 제안을 요청 편집기에 채우거나, cURL로 복사하거나, 제안 값을 환경 변수로 저장합니다.
  • 5 입력창 — Enter로 전송, Shift+Enter로 줄바꿈.

자연어로 요청 폼 채우기

/요청 판매중인 펫 목록을 조회해줘
/요청 사용자 ID가 42인 회원 정보를 조회해줘

AI는 요청 폼을 채워줄 뿐, 실제 전송은 사용자가 직접 ⌘Enter 또는 Send로 합니다. 의도치 않은 호출을 막기 위한 안전 장치입니다.

응답 설명 · 진단

요청을 보낸 뒤 Response 탭의 ✦ 설명 버튼(6장 2)을 누르면 응답 본문을 한국어로 해설해 줍니다. 4xx/5xx 오류면 원인과 해결 방향을 진단합니다. 커맨드 팔레트(⌘K)에서 “AI: 응답 설명/진단”으로도 실행할 수 있습니다.

대화 히스토리

대화는 프로젝트별로 자동 저장되어 앱을 재시작해도 복원됩니다.

보안 — 비밀 값은 AI에 전달되지 않습니다.

환경 변수의 값은 AI로 전송되지 않으며, 이름만 컨텍스트로 제공됩니다. 토큰·비밀번호 같은 민감 정보가 모델에 노출되지 않도록 설계되었습니다.

사전 조건 — 로컬 claude CLI 설치가 필요합니다.

AI 기능은 사용자의 컴퓨터에 설치된 claude CLI를 호출합니다. 앱은 시스템 PATH와 함께 ~/.local/bin, /opt/homebrew/bin (Windows: %USERPROFILE%\.local\bin, %APPDATA%\npm) 같은 일반적인 위치를 자동으로 탐지합니다. CLI를 찾지 못하면 AI 패널에 설치 안내가 표시됩니다.

claude CLI 설치

macOS / Linux

curl -fsSL https://claude.ai/install.sh | bash
claude            # 최초 1회 로그인

Windows (PowerShell)

irm https://claude.ai/install.ps1 | iex
claude            # 최초 1회 로그인

설치 후 SwaggerMan을 재시작하면 자동으로 claude를 찾습니다.

자동 탐지가 안 될 때: 설정(⚙) → “claude 실행파일 경로”에 경로를 직접 입력하세요. (예: macOS ~/.local/bin/claude, Windows C:\Users\<이름>\.local\bin\claude.exe)

12. 전역 헤더와 쿠키

전역 헤더

상단바 전역 헤더 설정(3장 11)에서 등록한 헤더는 이 프로젝트의 모든 요청에 자동으로 붙습니다. 공통 인증 키, 추적용 헤더 등에 사용하세요. 요청 편집기에는 읽기 전용으로 표시되어 어떤 헤더가 함께 나가는지 확인할 수 있습니다.

쿠키

서버가 내려준 쿠키(Set-Cookie)는 쿠키 저장소에 자동 보관되어 같은 호스트로 가는 다음 요청에 자동으로 첨부됩니다(브라우저처럼 동작). 세션 로그인이 필요한 API 테스트에 유용합니다.

  • 설정(⚙) → 쿠키 섹션에서 저장된 쿠키를 확인하고 모두 삭제할 수 있습니다.

13. 설정

상단바의 ⚙ 설정(3장 8)에서 네트워크·AI·쿠키를 관리합니다.

  • 1 네트워크 — 타임아웃(기본 30초), SSL 인증서 검증 무시(사내 CA·자체 서명 서버용), 프록시 URL(사내망에서 외부 API 호출 시). 스펙 로드·요청 실행·OAuth2 발급·업데이트 확인에 모두 적용됩니다.
  • 2 claude 실행파일 경로 — AI가 claude CLI를 자동으로 못 찾을 때 직접 지정합니다.
  • 3 쿠키 — 저장된 쿠키 조회·새로고침·모두 삭제.
  • 4 정보 — 현재 앱 버전 표시. 앱이 도움이 됐다면 카카오페이 QR로 개발자에게 커피 한 잔 ☕

테마와 화면 배율

  • 상단바 ☀︎/☾ 버튼으로 다크/라이트 테마를 전환합니다.
  • ⌘+ / ⌘− / ⌘0 또는 상단바의 −/+ 버튼으로 화면 배율을 조절합니다.

14. 자동 업데이트

SwaggerMan은 시작할 때 새 버전을 자동으로 확인합니다. 업데이트가 있으면 알림이 표시되고, 동의하면 자동으로 내려받아 설치한 뒤 재시작으로 최신 버전을 적용합니다. 상단바의 업데이트 확인 버튼으로 수동 확인도 가능합니다.

사내망(프록시 필요) 환경에서 “업데이트 확인 실패”가 뜨면 — 설정(⚙) → 네트워크 → 프록시 URL을 지정한 뒤 다시 확인하세요. 자동 확인이 실패하면 그 사유가 화면에 표시됩니다.

16. Mock 서버

로드한 스펙으로 로컬 가짜 API 서버를 띄워, 백엔드 없이 프론트엔드를 개발할 수 있습니다. 상단바 Mock 버튼으로 엽니다. 외부 클라이언트(브라우저·앱·curl)가 http://localhost:9090으로 직접 호출합니다.

  • 1 응답 데이터 3가지 소스 — 스키마 자동 생성(필드명 인식: 한국어 이름·이메일·날짜·금액 등, 시드 기반 일관성) / AI 생성(Claude가 도메인 맥락에 맞는 현실적 데이터) / 히스토리(실제 응답 재사용).
  • 2 진짜 백엔드처럼 — 목록↔단건 조회 일관성, page/size·offset/limit 페이징 자동, 응답 지연 시뮬레이션, CORS 허용.
  • 3 operation별 설정 — 활성/비활성, 상태코드·지연, 데이터셋 직접 편집(JSON).
  • 4 실시간 요청 로그 — 어떤 클라이언트가 무엇을 호출했는지 모달에서 확인.

17. 녹화 모드 (프록시 · 브라우저)

실제 API 호출을 녹화해 곧바로 Mock으로 저장합니다. 상단바 프록시 버튼으로 모달을 열고, 위쪽에서 프록시 / 브라우저 두 모드를 전환합니다.

프록시 모드 — 앱이 중계하며 녹화

실서버 앞에 로컬 프록시(localhost:9091)를 두고 트래픽을 투명 포워딩하면서 요청/응답을 자동 녹화합니다.

  • 1 타깃 Base URL · 포트를 지정하고 시작 → 프론트 앱을 localhost:9091로 향하게 하면 실서버로 전달되며 녹화됩니다. 설정(⚙)의 SSL 검증 끄기·프록시·타임아웃이 포워딩에도 적용돼 사내망(사설 CA) https 타깃도 녹화됩니다.
  • 2 실시간 녹화 목록 — 메서드·경로·상태코드가 쌓입니다.
  • 3 Mock으로 / 전체 Mock으로 — 항목별로, 또는 목록 전체를 한 번에 Mock으로 저장합니다. 같은 API가 여러 번 녹화됐으면 최신 응답이 저장되고, 스펙에 없는 경로는 제외 건수로 알려줍니다.

브라우저 모드 — Okta 등 로그인 흐름 녹화 v0.4.11

Okta 같은 OAuth/SSO 로그인은 인증이 끝나면 브라우저가 실제 서비스 호스트로 리다이렉트되어 프록시를 벗어납니다. 그래서 프록시 모드로는 로그인 뒤 트래픽을 잡을 수 없습니다. 브라우저 모드는 앱이 전용 Chrome 창을 띄워 그 브라우저 안의 API 호출(XHR/fetch)을 직접 녹화하므로, 로그인 중 호스트가 몇 번 바뀌어도 끝까지 따라가며 잡습니다.

  • 1 모달에서 브라우저 탭을 고르고 시작 URL(서비스 진입 주소)을 넣은 뒤 시작을 누릅니다.
  • 2 뜬 전용 Chrome에서 평소처럼 로그인하고 서비스를 사용하면, 호출된 API가 목록에 실시간으로 쌓입니다. 저장은 프록시 모드와 동일하게 Mock으로 / 전체 Mock으로.
  • 3 전용 프로필을 쓰므로 한 번 로그인하면 세션이 남아 다음 녹화 때 재로그인 없이 바로 잡힙니다.

정적 리소스·문서 이동은 빼고 API 호출(XHR/fetch)만 녹화합니다. 현재(v1)는 첫 탭만 캡처하므로, 로그인이 별도 팝업창으로 뜨는 방식의 SSO는 지원하지 않습니다(대부분의 OIDC는 같은 탭 리다이렉트라 문제없습니다).

18. API 성능 추이

상단바 성능 버튼 — 히스토리에 쌓인 응답시간을 operation별로 집계해 보여줍니다.

  • 1 통계 — 호출 수·평균·p95 응답시간. 헤더 클릭으로 정렬.
  • 2 스파크라인 — 시간순 응답시간 추이를 미니 차트로.
  • 3 느려짐 감지 — 최근 응답이 이전보다 뚜렷이 느려지면 ⚠️ 느려지는 중, 빨라지면 ✅로 표시합니다.

19. API 시간여행

상단바 시간여행 버튼 — 선택한 API의 응답을 주기적으로 스냅샷 저장하고 과거를 탐색합니다. "어제 이 API가 뭘 반환했지?", "장애 전후 응답이 어떻게 바뀌었지?"를 추적할 때 씁니다.

  • 1 대상 API 선택 + 지금 스냅샷(수동) 또는 자동 주기(1/5/15/30분, 앱이 열려 있는 동안).
  • 2 타임라인 — API별로 시간순 스냅샷을 보고, 클릭하면 그때의 응답 본문을 봅니다.
  • 3 두 스냅샷 비교 — 2개를 골라 비교 모달로 응답 차이(diff)를 확인합니다.

20. 가이드 문서 · 플로우 빌더

가이드 문서 생성

상단바 가이드 버튼 — 선택한 API들의 스펙(파라미터)과 히스토리의 실제 요청/응답 예시를 묶어 Markdown 연동 가이드를 만듭니다. 복사하거나 .md 파일로 저장해 동료에게 전달하세요. 요청 예시는 cURL로, 토큰 같은 민감 헤더는 자동 제외됩니다.

플로우 빌더

상단바 플로우 버튼 — API 단계를 세로로 나열해 순차 실행하는 시나리오를 만듭니다 (로그인 → 토큰 추출 → 생성 → 검증).

  • 1 단계 추가 — operation을 골라 단계를 쌓고, 드래그(≡)로 순서를 바꿉니다.
  • 2 추출 · 어서션 — 각 단계 응답에서 {{변수}}를 추출하면 다음 단계 요청에 자동으로 채워집니다. 단계별 상태/값 검증(어서션)도 가능합니다.
  • 3 전체 실행 — 순서대로 실행하며 단계별 상태코드·추출된 변수·어서션 통과(✅/❌)를 표시합니다.

21. 메모 · 공유 · 권한 매트릭스

API 메모 + 상태 태그

요청 화면 상단 📝 메모에 API별 메모와 상태(⚠️ Deprecated / 🔍 검토중 / ✅ 안정 / 🚫 사용금지)를 남깁니다. 사이드바에 상태 색상 점과 메모 아이콘으로 표시됩니다. 로컬에 저장됩니다.

요청 공유

상단바 공유 버튼 — 현재 요청을 압축 코드(swaggerman:req:…)로 복사해 동료에게 전달하고, 받는 쪽은 "가져오기"에 붙여넣어 적용합니다. 서버 없이 동작하며, 토큰·비밀번호 등 민감 헤더는 기본 제외됩니다.

권한 매트릭스

상단바 권한 버튼 — 토큰 여러 개(관리자·일반·게스트 등)로 선택한 API들을 호출해 권한별 접근 가능 여부를 상태코드 표로 비교합니다. 401·403은 주황으로 강조됩니다. GET이 기본 선택되며, 쓰기 요청(POST/PUT/DELETE)을 포함하면 실행 전 경고가 뜹니다.

22. 자주 묻는 질문

✦ AI 패널이 안 보이거나 동작하지 않아요.

AI 기능은 로컬 claude CLI가 있어야 동작합니다. 다음을 확인하세요.

  • 터미널에서 claude --version이 정상 실행되는지 확인
  • CLI가 PATH, ~/.local/bin, /opt/homebrew/bin 중 한 곳에 있는지 확인
  • 설치 후에도 인식되지 않으면 SwaggerMan을 재시작하거나, 설정(⚙)에서 경로를 직접 지정
내 토큰이나 비밀 값이 AI로 전송되나요?

아니요. 환경 변수의 값은 AI에 전달되지 않으며, 이름만 컨텍스트로 제공됩니다.

AI가 만든 요청이 자동으로 전송되나요?

아니요. AI는 폼을 채워줄 뿐이며, 실제 전송은 사용자가 ⌘Enter 또는 Send로 직접 합니다.

macOS에서 “개발자를 확인할 수 없습니다”라고 떠요.

Finder에서 SwaggerMan 아이콘을 우클릭 → 열기 후 다시 열기를 누르면 됩니다. 안 되면 터미널에서 xattr -dr com.apple.quarantine /Applications/SwaggerMan.app을 실행하세요.

스펙을 불러올 수 없어요.

URL이 올바른 OpenAPI(JSON/YAML) 스펙을 가리키는지, 네트워크 접근이 가능한지 확인하세요. 사내 CA·자체 서명 인증서 서버라면 설정(⚙) → SSL 인증서 검증 무시를 켠 뒤 다시 시도하세요.

요청 샘플과 컬렉션은 뭐가 다른가요?

요청 샘플은 한 API 안에서 Query·Headers·Body 묶음을 여러 세트로 보관하고 전환하는 기능입니다(예: 같은 API의 정상 케이스/에러 케이스). 컬렉션은 여러 API의 요청을 모아 두고 일괄 실행(러너)하거나 Postman과 주고받는 보관함입니다.

앱을 재시작했는데 입력하던 값이 사라졌어요.

파라미터·헤더·Body 입력값은 API별로 자동 저장되어 재시작 후 복원됩니다. 응답 본문은 복원되지 않지만, 히스토리에서 과거 요청·응답을 다시 볼 수 있습니다.

사내망(프록시 환경)에서 업데이트 확인이 실패해요.

설정(⚙) → 네트워크 → 프록시 URL을 지정한 뒤 다시 시도하세요. 프록시 설정은 스펙 로드·요청 실행·업데이트 확인에 모두 적용됩니다.

대화 내용이 사라졌어요.

AI 대화는 프로젝트별로 자동 저장·복원됩니다. “새 대화”를 눌렀거나 다른 프로젝트로 전환하면 해당 프로젝트의 히스토리가 표시됩니다.
